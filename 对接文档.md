API 接口说明
本文档与当前用户网页调用方式一致：使用服务商提供的 Base URL（与网页配置的后端地址相同），以及 API Key（sk-…）调用下文 /v1 等路径。

Base URL：由商务/部署提供；网页内绑定 Key 后，所有请求发往同一后端根地址。路径均为相对路径，拼在 Base URL 之后（注意末尾不要重复 /）。
鉴权
除 GET /health 外，业务接口均需携带 API Key（与网页「绑定 API Key」相同）：

Header	值
Authorization	Bearer <你的 sk- API Key>
提交 multipart/form-data 时不要手动设置 Content-Type（由客户端自动生成 boundary）。

错误码与响应
HTTP	说明
400	参数错误
401	鉴权失败（Key 无效、禁用等）
402	余额不足
429	限流或并发限制；生成中任务过多时可能返回 code: GENERATING_LIMIT
500 / 503	服务端异常或繁忙
错误体常见为 { "error": "字符串" } 或 { "error": { "message": "…" } }，以实际响应为准。

GET
健康检查
GET /health
网页绑定 Key 时用此接口校验 Key 是否可用；仅需 Key 时可带 Authorization: Bearer …。

POST
视频生成
POST /v1/videos/generations
Content-Type: multipart/form-data。与网页「视频创作」提交字段一致，核心字段如下（未列出的可选字段以实现为准）：

字段	类型	说明
prompt	string	提示词（必填）
model	string	如 seedance-2.0、seedance-2.0-fast
ratio	string	宽高比
duration	number	时长（秒），网页约束一般为 4~15
video_resolution	string	如 720p、1080p
channel	string	渠道（与网页下拉一致）
function_mode	string	如 first_last_frames、first_frame、end_frame、omni_reference、multi_frame 等
first_frame / end_frame	file	首尾帧图片文件
frame_1…	file	多帧参考模式下的帧文件
image_file_1…、video_file_1…、audio_file_1…	file	混合素材（全能引用）模式
materials	string	JSON 字符串：引用素材库时由网页组装
成功时响应含 task_id（UUID 字符串）、status（如 pending）、model、created_at 等，与网页轮询所用一致。

POST
画质增强（与网页一致）
POST /v1/videos/enhance
网页在完成一条视频生成任务并取得成片地址后，可对指定输出 URL 发起「画质增强」子任务。Content-Type: application/json，需携带与视频生成相同的 Authorization: Bearer <API Key>。

调用前提
已通过 GET /v1/tasks/{source_task_id}（或任务列表）确认源任务状态为成功，且 result_urls 非空。
url_index 指向 result_urls 中要增强的那一条（多输出时从 0 开始计数）。
请求体字段
与网页内「画质增强」按钮提交逻辑一致（字段名使用下划线）：

字段	类型	必填	说明
source_task_id	string	是	源视频任务的 task_id（UUID）
url_index	number	否	使用 result_urls 的第几条，默认 0
tool_version	string	否	网页可选 standard（标准）或 professional（专业），默认与页面「标准」一致
scene	string	否	场景标识，网页默认 aigc，以后台支持的枚举为准
resolution	string	否	目标分辨率，网页默认 1080p，可选值以后台为准
fps	number	否	目标帧率，网页默认 30，可选如 60 等
成功响应
HTTP 成功时返回 JSON，其中包含新的增强任务 task_id（与源任务 ID 不同）。网页将该 ID 写入本地任务缓存并进入轮询。

轮询与结果
与网页逻辑一致：使用 GET /v1/tasks/{task_id}（上一步返回的 task_id）定期查询，建议间隔约 3 秒。

status（示例）	说明
success	增强完成，从 result_urls 取成片地址
failed	失败，查看 fail_reason
pending / submitted / generating / post_processing	排队或处理中；网页会用 progress_text、progress_pct（若存在）更新进度展示
增强任务在列表中会体现为关联源任务（例如带 source_task_id 等字段，具体以接口返回为准）。

复制

GET
任务详情
GET /v1/tasks/{task_id}
网页工作区、视频生成与画质增强轮询均使用此接口。响应含 status、result_urls、fail_reason、progress、progress_text、progress_pct、model、task_type 等字段（以实际 JSON 为准）。

POST
任务列表
POST /v1/tasks/list
JSON 请求体：

字段	说明
page	页码，默认 1
page_size	每页条数（网页任务列表用 20，工作区刷新用 50）
status	可选，筛选状态（如 success / failed）
响应含 items（任务摘要数组）、total 等。

GET
账户余额
GET /v1/balance
网页账户区使用的只读接口；返回字段名与 JSON 结构以实际响应为准（常见含 balance、frozen、customer_id 等）。

GET
账户流水
GET /v1/transactions
查询参数：page、page_size（网页固定 20）、type（可选，用于筛选类型）。响应含 transactions、total、page_size；单条字段以后端为准。

GET
模型列表
GET /v1/models
响应形如 { "data": [ { "id", "type" }, ... ] }，网页用其中的视频模型填充下拉框。

GET
渠道配置
GET /v1/channels
网页在部分场景下用于拉取渠道侧配置供界面使用；响应为 JSON 对象，字段名与层级随后端版本可能变化，对接时请以实际响应为准。

素材库
与网页素材面板一致

POST
/v1/asset-library/assets
multipart/form-data：file（必填）、可选 name、thumbnail（Data URL）。

POST
/v1/asset-library/assets/list
JSON：filter（可选 asset_type、status）、page_number、page_size；响应含 items、total_count。

DELETE
/v1/asset-library/assets/{id}
删除指定素材 ID。

调用流程
视频生成：① POST /v1/videos/generations 得到 task_id → ② 轮询 GET /v1/tasks/{task_id} 至 success / failed → ③ 自 result_urls 取地址。
画质增强（在源视频已成功之后）：① POST /v1/videos/enhance（JSON 体含 source_task_id 等）→ ② 用返回的新 task_id 同样轮询 GET /v1/tasks/{task_id}（建议间隔约 3 秒）→ ③ 成功时新的成片在 result_urls。
示例：提交视频生成（curl）
Shell
复制
curl -X POST "${API_BASE}/v1/videos/generations" \
  -H "Authorization: Bearer ${API_KEY}" \
  -F "prompt=一只猫咪在草地上奔跑" \
  -F "model=seedance-2.0" \
  -F "ratio=16:9" \
  -F "duration=8" \
  -F "video_resolution=720p" \
  -F "channel=<与网页「渠道」一致的取值>"
示例：提交画质增强（curl）
Shell
复制
curl -X POST "${API_BASE}/v1/videos/enhance" \
  -H "Authorization: Bearer ${API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{"source_task_id":"源任务UUID","url_index":0,"tool_version":"standard","scene":"aigc","resolution":"1080p","fps":30}'
